## PASO 2 — Crear script de seed

Crear el archivo `backend/app/db/seed.py` con este patrón exacto:
```python
import asyncio
from sqlalchemy import delete
from app.db.session import AsyncSessionLocal
from app.models.{dominio} import {Dominio}
# importar un modelo por cada tabla a poblar

async def seed_{dominio}(db):
    """Borra e inserta datos de {dominio}"""
    await db.execute(delete({Dominio}))
    items = [
        {Dominio}(
            campo1="valor extraído del frontend.md",
            campo2="valor extraído del frontend.md",
            image_url=None,  # las imágenes se suben por Cloudinary, no en seed
        ),
        # un objeto por cada registro
    ]
    db.add_all(items)

async def main():
    async with AsyncSessionLocal() as db:
        # ejecutar en orden para respetar dependencias
        await seed_{dominio1}(db)
        await seed_{dominio2}(db)
        # una llamada por cada función de seed
        await db.commit()
        print("✅ Seed completado")

if __name__ == "__main__":
    asyncio.run(main())
```

Reglas del seed:
- `DELETE + INSERT` siempre — nunca `INSERT OR IGNORE` ni `UPSERT`
- `image_url=None` en todos los registros — las imágenes se gestionan
  por Cloudinary desde el panel de administración, no desde el seed
- Un `seed_{dominio}` por tabla, no mezclar dominios en la misma función
- Los valores deben venir del `frontend.md`, no inventarlos

## Reglas absolutas

- ❌ Nunca crear modelos, schemas ni endpoints en este prompt
- ❌ Nunca inventar datos — todo viene del `frontend.md`
- ❌ Nunca usar `INSERT OR IGNORE`, `UPSERT` ni `ON CONFLICT` —
  siempre `DELETE + INSERT`
- ❌ Nunca seedear `image_url` con rutas locales o URLs de Unsplash —
  dejarlo `None`, las imágenes van por Cloudinary
- ❌ Nunca mezclar lógica de dos dominios en la misma función de seed
- ❌ Nunca ejecutar el seed sin mostrar primero la tabla del Paso 1
  y recibir confirmación
- ✅ Si un campo del frontend.md no tiene columna en la tabla, ignorarlo
- ✅ Si hay datos de configuración técnica (colores, fuentes) que tienen
  tabla destino, incluirlos en el seed
- ✅ Ante cualquier duda de mapeo entre dato y columna, preguntar
  antes de insertar
